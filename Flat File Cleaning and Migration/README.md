# Flat File Cleaning and Migration

This project cleans a raw CSV of email subscribers, exports a validated flat file, captures rejected rows, and can optionally load the cleaned data into PostgreSQL.

## What Improved

- The workflow is now self-contained inside this folder.
- `load.py` cleans, validates, deduplicates, exports, and optionally loads data.
- Output files are written with stable project-local paths instead of relying on a missing external CSV.
- Rejected rows are preserved with a rejection reason for auditability.
- Database loading is configurable through `DATABASE_URL` or CLI arguments instead of a hard-coded password.

## Files

- `raw_email_list.csv`: raw source data
- `clean_email_list.csv`: cleaned output generated by the pipeline
- `rejected_email_records.csv`: invalid or duplicate rows generated by the pipeline
- `exported.sql`: PostgreSQL table definition
- `load.py`: ETL script

## Run The Pipeline

```bash
python load.py
```

This will:

1. Read `raw_email_list.csv`
2. Standardize names, email casing, countries, and dates
3. Remove duplicate emails by keeping the earliest signup
4. Export cleaned and rejected records

## Load Into PostgreSQL

Set a connection string, then run:

```bash
python load.py --load-to-db
```

You can also pass the URL directly:

```bash
python load.py --load-to-db --database-url "postgresql://postgres:password@localhost:5432/email_db"
```

## Output Rules

- Blank names become `Unknown`
- Blank countries become `Unknown`
- `USA` becomes `United States`
- `UK` becomes `United Kingdom`
- Invalid emails are rejected
- Missing signup dates are rejected
- Duplicate emails are rejected after the earliest valid signup is kept
